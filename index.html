CI/CD Pipeline, Concourse

  • Git/Github repository
  • CI: Each check-in is then verified by an automated build
  • Build: compiles code, runs unit/integration tests, creates and stores the docker image
  • CD: an approach in which teams ensure that every change to the system is releasable, and that we can release any version at the push of a button
  • Deploys the docker image into QA(acceptance testing), Load(Load testing), PROD environments
  • Creates DBs, Kafka Topic
  • Every CD pipeline is unique
  • Key for DevOps
  • Concourse

Docker/Kubernetes

  • Docker is a shipping container system for code
  • Build once (finally) run anywhere
  • Containers vs VMs: Containers are isolated but share the same OS kernel, lightweight, faster start up(seconds)
  • Containers and VMs together provide a tremendous amount of flexibilidy to deploy and manage apps

Docker/Kubernetes

  • Kubernetes: Container Orchestrator/Cluster Manager
  • Developed by Google
  • Places containers on nodes
  • Recovers automatically from failure
  • Basic monitoring, logging, health checking
  • Enables containers to find each other
  • kubernetes-101/

gRPC

  • Multiplatform RPC system
  • Developed by Google
  • Allow sync/async communication
  • Based on http/2 Http demo
  • C, C++, Java, Scala, C#, etc
